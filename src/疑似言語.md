# 疑似言語 学習ガイド

本ガイドは、ITパスポートや基本情報技術者試験の合格を目指す学習者向けに、アルゴリズム問題で用いられる「疑似言語」の文法と概念を体系的に解説します。確認問題を通じて、実践的な読解力と論理的思考力を養うことを目的としています。

## 第1章 疑似言語の基礎

### 1. 疑似言語とは

疑似言語は、特定のプログラミング言語に依存せず、アルゴリズムの論理構造を表現するために設計された記述言語です。基本情報技術者試験では、午後のアルゴリズム問題において、かつて主流だった「流れ図（フローチャート）」に代わり導入されました。

これは、現代のプログラミングにおける「構造化プログラミング」の考え方を反映したもので、アルゴリズムを以下の3つの基本構造のみを組み合わせて表現します。

* 順次 (Sequential): 記述された順に処理を実行する構造。
* 判断 (Selection/Conditional): 条件に応じて処理の流れを分岐させる構造。
* 反復 (Iteration/Loop): 特定の条件が満たされる間、処理を繰り返し実行する構造。

### 2. 疑似言語の全体構成

疑似言語で記述されたプログラムは、大きく「宣言部」と「処理部」の2つの部分から構成されます。

|部|説明|
|---|---|
|宣言部|プログラムの名称、およびプログラム内で使用する変数、配列、レコードなどのデータ構造を定義します。|
|処理部|アルゴリズムの具体的な処理内容を、順次・判断・反復の構造を組み合わせて記述します。|

コメント（注釈）

プログラムの動作には影響を与えない説明文を記述するために使用します。コードの可読性を高める目的で用いられます。

/* この中にコメントを記述します */
// この行の以降がコメントになります


## 第2章 宣言部の詳細

宣言部では、プログラム名や、処理で使用するデータの「器」となる変数を定義します。定義は、行頭に「○」を付けて記述します。

### 1. プログラム名称

プログラムの名称を定義します。

記述例: ○合計を求める

### 2. 変数とデータ型

変数はデータを格納するための名前付きの領域です。使用する前に、どのような種類のデータ（データ型）を扱うかを宣言する必要があります。

主なデータ型

|データ型|説明|記述例|
|---|---|---|
|整数型|整数（小数点を含まない数）を扱う。|-5, 0, 10, 123|
|実数型|実数（小数点を含む数）を扱う。|1.25, -52.325, 3.14|
|文字型|1文字のデータを扱う。|'A', 'い', 'ぬ'|
|文字列型|複数の文字が連なった文字列を扱う。|'PAFUTAMA', 'いぬぼき'|
|論理型|「真 (true)」または「偽 (false)」の2値のみを扱う。|true, false|

変数の宣言

記述例1: 単一の変数を宣言 整数を格納する変数 sum を宣言する場合。 ○整数型：sum

記述例2: 複数の変数を宣言 2つの実数を格納する変数 A と B を宣言する場合。カンマ (,) で区切ります。 ○実数型：A, B

### 3. 配列の定義

配列は、同じデータ型の複数のデータを連続的に並べたデータ構造です。各データは「要素」と呼ばれ、添字（インデックス） と呼ばれる番号で区別されます。

注意点: 配列の添字が「0」から始まるか「1」から始まるかは、試験の問題文で指定されます。必ず確認してください。

記述例1: 要素数を指定して宣言 100個の整数型要素を持つ配列 sum を宣言する場合。 ○整数型：sum[100]

記述例2: 初期値を設定して宣言 初期値 (1, 2, 3, 4, 5) を持つ整数型配列 sum を宣言する場合。要素数は自動的に5となります。 ○整数型：sum[] = (1, 2, 3, 4, 5)

記述例3: 2次元配列の宣言 10行5列の文字型2次元配列 moji を宣言する場合。 ○文字型：moji[10][5]

4. レコード型（構造体）の定義

レコード型は、複数の異なるデータ型の変数を一つのまとまりとして扱うためのデータ型です。「構造体」とも呼ばれます。レコード内の各変数はピリオド (.) で区切って指定します。

記述例1: レコード型の定義 生徒の個人情報（生徒番号、氏名、電話番号）をまとめたレコード 生徒 を定義する場合。 ○レコード：生徒 (整数型：生徒番号. 文字型：氏名[20]. 文字型：電話番号[12])

記述例2: レコード型の配列定義 上記の 生徒 レコードを100人分扱うための配列を定義する場合。 ○レコード：生徒[100] (整数型：生徒番号. 文字型：氏名[20]. 文字型：電話番号[12])

第3章 処理部の詳細

処理部では、アルゴリズムの具体的な手順を記述します。処理の各行は、先頭に「・」を付けて記述されることがありますが、省略される場合もあります。

1. 順次構造と代入

処理は上から下へ順番に実行されます。最も基本的な処理は代入です。

・A ← B

これは「変数Bの内容（値）を変数Aに格納する」ことを意味します。流れ図の矢印 (→) とは向きが逆である点に注意してください。

算術演算子

計算には以下の演算子が使用されます。

演算子	意味	記述例
+	加算	M ← M + 1 (Mの値を1増やす)
-	減算	A ← B - C
*	乗算	S ← H * W
/	除算	Avg ← Sum / N
%	剰余	Amari ← X % Y (XをYで割った余り)

2. 判断構造 (if文)

条件式の結果（真または偽）に応じて処理を分岐させます。if から始まり endif で終わります。

条件演算子と論理演算子

種類	演算子	意味	記述例
条件演算子	=	等しい	A = 10
	≠	等しくない	A ≠ B
	>	より大きい	Score > 80
	<	より小さい	Age < 20
	≧	以上	A ≧ 100
	≦	以下	A ≦ 200
論理演算子	and	論理積（かつ）	A ≧ 100 and A ≦ 200
	or	論理和（または）	A < 0 or A > 100
	not	否定	not (A = B)

判断構造の記述形式

形式1: 基本形 (if) もし条件式が真ならば、処理を実行します。

if (x ≧ y)
  ans ← x
endif


形式2: 分岐 (if-else) もし条件式が真ならば処理1を、偽ならば処理2を実行します。

if (x ≧ y)
  ans ← x
else
  ans ← y
endif


形式3: 多分岐 (if-elseif-else) 複数の条件で分岐します。上から順に評価され、最初に真となった条件の処理のみが実行されます。

if (age が 3 以下)
  ret ← 100
elseif (age が 9 以下)
  ret ← 300
else
  ret ← 500
endif


3. 反復構造 (ループ)

特定の処理を繰り返し実行するための構造です。

① 前判定反復 (while文)

ループ内の処理を実行する前に条件式を評価します。条件式が真である間、処理を繰り返します。最初に条件が偽の場合、ループ内の処理は一度も実行されません。

注意点: 流れ図の反復が「終了条件」を記述するのに対し、疑似言語のwhile文は「継続条件」を記述します。

記述例:

// iが5以下の間、処理を繰り返す
while (i ≦ 5)
  // 処理
  i ← i + 1
endwhile


② 回数が決まっている反復 (for文)

繰り返す回数が明確な場合に使用されます。カウンタ変数の初期値、継続条件、増分をまとめて記述します。

記述例:

// iを1から5まで1ずつ増やしながら繰り返す
for (iを1から5まで1ずつ増やす)
  ans ← ans + i
endfor


別の記述形式として、以下のように示されることもあります。 ■ i ：1, i <= 100, 1

③ 後判定反復 (do-while文)

ループ内の処理を一度実行した後に条件式を評価します。そのため、ループ内の処理は最低1回は必ず実行されます。試験での出題頻度は低いとされています。

記述例:

do
  // 処理
while (条件式)


第4章 関数

関数は、特定のまとまった処理を実行するプログラム部品です。「副プログラム」「メソッド」「プロシージャ」「サブルーチン」などとも呼ばれます。近年の試験では、関数形式での出題が主流です。

1. 関数の定義

関数は、「関数名」「引数（ひきすう）」「戻り値（もどりち）」の3つの要素で定義されます。

記述形式: ○戻り値のデータ型：関数名（引数リスト）

* 戻り値: 関数の処理結果として呼び出し元に返される値。戻り値がない場合、データ型の記述は省略されます。
* 引数リスト: 関数が処理を行うために外部から受け取る値。データ型：変数名 の形式で記述し、複数ある場合はカンマ (,) で区切ります。

記述例1: 戻り値と引数がある関数 ○整数型：Select（整数型の配列：x, 整数型：n）

* 関数名: Select
* 戻り値: 整数型の値
* 引数: 整数型の配列 x と、整数型の変数 n

記述例2: 戻り値がない関数（手続） ○proc1()

* 関数名: proc1
* 戻り値: なし
* 引数: なし

2. 関数の呼び出しと引数

定義した関数は、プログラムの処理部から呼び出すことで実行されます。

* 仮引数 (parameter): 関数を定義する際に記述される引数。
* 実引数 (argument): 関数を呼び出す際に実際に渡す値や変数。

記述例:

// Kensaku関数を呼び出し、戻り値を変数retに代入する
ret = Kensaku(HAI, 77)


この例では、HAI と 77 が実引数です。

3. 関数で使われる重要な命令

命令	説明
return	関数の処理を終了し、呼び出し元に戻り値を返します。戻り値がある関数では必須です。
break	反復構造（ループ）の処理を強制的に中断し、ループから脱出します。

問1 次のプログラムにおいて、手続 proc2 を呼び出すと、 “C”，“B”，“A”，“C” の順に出力される。 (○か×か)

【プログラム】

○proc1()
  ”A” を出力する
  proc3()

○proc2()
  proc3()
  ”B” を出力する
  proc1()

○proc3()
  ”C” を出力する


答え：○

解説: 処理の実行順序（トレース）は以下のようになります。

1. proc2() を呼び出す。
2. proc2 内の proc3() が呼び出される。
3. proc3 が実行され、"C" が出力される。
4. proc2 に戻り、次の行の "B" を出力する が実行され、"B" が出力される。
5. proc2 内の proc1() が呼び出される。
6. proc1 内の "A" を出力する が実行され、"A" が出力される。
7. proc1 内の proc3() が呼び出される。
8. proc3 が実行され、"C" が出力される。

したがって、出力順は "C", "B", "A", "C" となります。

問2 次のプログラムを実行すると、”3,2″と出力される。 (○か×か)

【プログラム】

整数型：x ← 1
整数型：y ← 2
整数型：z ← 3
x ← y
y ← z
z ← x
yの値とzの値をこの順にコンマ区切りで出力する


答え：○

解説: 変数の値の移り変わりをトレース表で確認します。

手順	x	y	z	説明
初期値	1	2	3	
① x ← y	2	2	3	xにyの値(2)を代入
② y ← z	2	3	3	yにzの値(3)を代入
③ z ← x	2	3	2	zにxの値(2)を代入

最終的に、y の値は 3、z の値は 2 となります。これをコンマ区切りで出力すると "3,2" となります。

問3 ある施設の入場料は、0 歳から 3 歳までは 100 円、4 歳から 9 歳までは 300 円、10 歳以上は 500 円である。関数 fee は、年齢を表す 0 以上の整数を引数として受け取り、入場料を返す。プログラム中の ( ？ ) に入る適切な字句は「age が 9 以下」である。 (○か×か)

【プログラム】

○整数型: fee(整数型: age)
  整数型: ret
  if (age が 3 以下)
    ret ← 100
  elseif ( ？ )
    ret ← 300
  else
    ret ← 500
  endif
  return ret


答え：○

解説: if-elseif-else 構文は上から順に条件が評価されます。

1. 最初の if (age が 3 以下) で、3歳以下の場合は ret ← 100 となり、その時点で判断構造は終了します。
2. この条件に一致しなかった場合（つまり、ageが4歳以上の場合）に、次の elseif ( ？ ) の条件が評価されます。
3. elseif のブロックでは ret ← 300 となっているので、この条件は「4歳から9歳まで」に該当する必要があります。
4. すでに「3歳以下ではない」（＝4歳以上である）ことは確定しているため、 ( ？ ) には残りの条件である「9歳以下」を指定すれば十分です。

したがって、( ？ ) には age が 9 以下 が入ります。

問4 関数 checkDigit は、10進9桁の整数の各桁の数字が上位の桁から順に格納された整数型の配列 originalDigit を引数として、次の手順で計算したチェックデジットを戻り値とする。プログラム中の [　？　] に入る適切な字句は「j ← k＋( j－10×k )」である。ここで、配列の要素番号は1から始まる。 (○か×か)

〔手順〕 (1) 配列 originalDigit の要素番号1〜9の要素の値を合計する。 (2) 合計した値が9より大きい場合は、合計した値を10進の整数で表現したときの各桁の数字を合計する。この操作を、合計した値が9以下になるまで繰り返す。 (3) (2)で得られた値をチェックデジットとする。

〔プログラム〕

○整数型：checkDigit（整数型の配列：originalDigit）
  整数型：i, j, k
  j ← 0
  for（iを1からoriginalDigitの要素数まで1ずつ増やす）
    j ← j ＋ originalDigit［i］
  endfor
  while（jが9より大きい）
    k ← j ÷ 10 の商　/* 10進9桁の数の場合、jが2桁を超えることはない */
    [　？　]
  endwhile
  return j


答え：○

解説: while ループは、手順(2)の「合計した値 j が9より大きい場合に、j の各桁の数字を合計して新しい j とする」処理を実装しています。

* k ← j ÷ 10 の商 は、2桁の数 j の10の位の数字を計算しています。 (例: j が58の場合、58 ÷ 10 の商は 5 となり、k に 5 が代入されます)
* 次に必要なのは1の位の数字です。1の位は、元の数 j から「10の位の数 × 10」を引くことで求められます。 (例: j が58の場合、58 - (5 * 10) = 8 となり、8 が1の位の数字です)
* 手順(2)に従い、この10の位の数字 (k) と1の位の数字 (j - 10 * k) を合計したものが、新しい j となります。
* よって、 j ← k + (j - 10 * k) という式が導かれます。

したがって、[　？　] には j ← k＋( j－10×k ) が入ります。

問5 関数 calcX と関数 calcY は、引数 inData を用いて計算を行い、その結果を戻り値とする。関数 calcX を calcX(1) として呼び出すと、関数 calcX の変数 num の値が、1 → 3 → 7 → 13 と変化し、戻り値は13となった。関数 calcY を calcY(1) として呼び出すと、関数 calcY の変数 num の値が、1 → 5 → 13 → 25 と変化し、戻り値は25となった。 プログラム中の a, b に入れる字句の適切な組合せは a: num ＋ 2 × i, b: iを2から6まで2ずつ増やす である。 (○か×か)

【プログラム1】

○整数型：calcX(整数型：inData)
  整数型：i, num
  num ← inData
  for(iを1から3まで1ずつ増やす)
    num ← [ a ]
  endfor
  return num


【プログラム2】

○整数型：calcY(整数型：inData)
  整数型：i, num
  num ← inData
  for([ b ])
    num ← num ＋ 2 × i
  endfor
  return num


答え：○

解説:

[プログラム1: calcX のトレース] calcX(1) を呼び出すと、初期値 num は 1。i は 1, 2, 3 と変化します。a に num ＋ 2 × i を入れて計算します。

* i=1 のとき: num ← 1 + 2 * 1 = 3
* i=2 のとき: num ← 3 + 2 * 2 = 7
* i=3 のとき: num ← 7 + 2 * 3 = 13 num の値は 1 → 3 → 7 → 13 と変化し、問題文と一致します。

[プログラム2: calcY のトレース] calcY(1) を呼び出すと、初期値 num は 1。b に iを2から6まで2ずつ増やす を入れると、i は 2, 4, 6 と変化します。

* i=2 のとき: num ← 1 + 2 * 2 = 5
* i=4 のとき: num ← 5 + 2 * 4 = 13
* i=6 のとき: num ← 13 + 2 * 6 = 25 num の値は 1 → 5 → 13 → 25 と変化し、問題文と一致します。

したがって、指定された組合せは正しいです。

問6 関数 makeNewArray は、要素数2以上の整数型の配列を引数にとり、整数型の配列を返す関数である。関数 makeNewArray を makeNewArray({3, 2, 1, 6, 5, 4}) として呼び出したとき、戻り値の配列は {3, 5, 6, 12, 17, 21} となる。ここで、配列の要素番号は1から始まる。 (○か×か)

〔プログラム〕

◯整数型の配列：makeNewArray(整数型の配列： in)
  整数型の配列：out ← { }　// 要素数0の配列
  整数型： i, tail

  outの末尾にin[1]の値を追加する
  for (iを2からinの要素数まで1ずつ増やす)
    tail ← out[outの要素数]
    outの末尾に(tail + in[i])の結果を追加する
  endfor
  return out


答え：○

解説: プログラムの動作をトレースします。

* 初期状態: in = {3, 2, 1, 6, 5, 4}, out = {}
* outの末尾にin[1]の値を追加する: in[1] は 3 なので、out は {3} となります。
* for ループ開始:
  * i=2:
    * tail ← out[outの要素数] → outの要素数は1なので tail ← out[1] → tail は 3。
    * outの末尾に(tail + in[2])の結果を追加 → 3 + 2 = 5 を追加。out は {3, 5}。
  * i=3:
    * tail ← out[outの要素数] → outの要素数は2なので tail ← out[2] → tail は 5。
    * outの末尾に(tail + in[3])の結果を追加 → 5 + 1 = 6 を追加。out は {3, 5, 6}。
  * i=4:
    * tail ← out[outの要素数] → outの要素数は3なので tail ← out[3] → tail は 6。
    * outの末尾に(tail + in[4])の結果を追加 → 6 + 6 = 12 を追加。out は {3, 5, 6, 12}。
  * i=5:
    * tail ← out[outの要素数] → outの要素数は4なので tail ← out[4] → tail は 12。
    * outの末尾に(tail + in[5])の結果を追加 → 12 + 5 = 17 を追加。out は {3, 5, 6, 12, 17}。
  * i=6:
    * tail ← out[outの要素数] → outの要素数は5なので tail ← out[5] → tail は 17。
    * outの末尾に(tail + in[6])の結果を追加 → 17 + 4 = 21 を追加。out は {3, 5, 6, 12, 17, 21}。
* return out: 最終的な配列 {3, 5, 6, 12, 17, 21} を返します。

したがって、戻り値は問題文と一致します。

第6章 用語集

|用語|説明|
|---|---|
|疑似言語|アルゴリズムの理解を助けるために用いられる、特定の文法に縛られない擬似的なプログラム言語。|
|構造化プログラミング|プログラムの構造を「順次」「判断」「反復」の3つの基本論理構造の組み合わせで構築する手法。|
|宣言部|プログラムで使用する変数や関数などをあらかじめ定義する部分。|
|処理部|アルゴリズムの具体的な処理内容を記述する部分。|
|データ型|変数がどのような種類のデータ（整数、実数、文字など）を扱うかを示す分類。|
|変数|プログラム中でデータを一時的に格納するための、名前が付けられた記憶領域。|
|配列|同じデータ型の複数のデータを、連続したメモリ領域に格納したデータ構造。|
|要素|配列を構成する個々のデータ。|
|添字 (インデックス)|配列の各要素を識別するための番号。|
|レコード型 (構造体)|複数の異なるデータ型の変数を一つにまとめたデータ型。|
|代入|変数に値を格納すること。疑似言語では ← 記号が使われる。|
|判断構造|条件式の結果（真偽）によって、実行する処理を分岐させる制御構造。if文が代表的。|
|反復構造 (ループ)|ある条件が満たされている間、特定の処理を繰り返し実行する制御構造。while文やfor文などがある。|
|関数|あるまとまった処理を行うプログラムの部品。呼び出して使用する。副プログラム、メソッド、プロシージャとも呼ばれる。|
|引数 (ひきすう)|関数を呼び出す際に、関数に渡す値やデータ。|
|仮引数|関数を定義する際に宣言される引数のこと。|
|実引数|関数を実際に呼び出す際に渡す値や変数のこと。|
|戻り値 (返却値)|関数の処理が終了した際に、呼び出し元に返される結果の値。|
|return文|関数の実行を終了し、戻り値を呼び出し元に返すための命令。|
|break文|反復構造（ループ）を途中で強制的に抜け出すための命令。|
|トレース|プログラムの実行順序に従って、変数の値の変化や処理の流れを一つずつ追跡すること。|
